# C++ primer

## 15 面向对象程序设计

### OPP 
- 核心
    1. 数据抽象：将类的接口和实现分离
    2. 继承：将类形成层次关系，根部为**基类**，继承得到的类为**派生类**
        - 虚函数：基类希望派生类自己定义适合的版本时，将函数声明为 <br/>
            `virtual void func();`
        - 类派生列表：明确指出是由哪些基类派生而来的,每个基类前可以有访问说明符 <br/>
          派生类必须对所有重新定义的虚函数进行声明
            ```
            class C1 : public C2, public C3 {
                void func() override; // C++11新标准
            };
            ``` 
    3. 动态绑定：当使用基类的引用，调用虚函数时，在运行时选择函数的版本，即选择执行基类的版本，还是派生类的版本

### 定义

#### 基类定义
- 成员函数
    1. virtual只能出现在类内部的声明语句，而不能用于类外部
    2. 虚函数使用动态绑定，且在派生类中，隐式的也是虚函数；而非虚函数解析过程在编译阶段
- 访问控制
    - 派生类只能访问 public 成员，而不能访问 private 成员
    - 基类中声明为 **protected**，代表派生类可以访问，而其他用户不可访问
